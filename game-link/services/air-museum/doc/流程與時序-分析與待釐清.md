# 流程與時序－分析與待釐清（已合併）

本文件所列之待釐清項目已併入 **[流程與時序.md](流程與時序.md)**，並依產品決策完成約定，摘要如下：

- 約定投影機先連，不會出現玩家比投影機先連。
- gameStateEvent 由主機 push 觸發，廣播對象為所有 player（不含主機）。
- 滿員時無特別回傳，手柄依 gameStateEvent 判斷啟用手柄或等待入桌。
- leave 不允許主機呼叫。
- playerProfile 在該玩家 **entry 並且通過之後** 由 Service 推送給投影端。
- 主機斷線重連後，Server 將所有玩家踢掉並關閉連線，讓手柄重新連回、重新 entry。
- input 僅接受房內玩家。
- pushGameState 僅主機可送。
- auth 錯誤直接回傳錯誤，手柄顯示再連線一次視窗。
- DB 查無玩家時，gameStateEvent 回傳給該玩家請其重新建立個人檔案。
- 時序圖已補「Server → 投影端：playerProfile」。
- maxPlayers 已補在 config；重複 entry 允許、視為已入桌；rid 已刪除。

詳細內容請見 **流程與時序.md**。
